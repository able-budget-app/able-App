Scale thresholds what changes at each band

v1 0–1k users

  • Supabase Pro ($25). Free tier won't cut it past launch — Pro gives daily backups + 8GB DB.
  • Netlify Pro ($20) — 1TB bandwidth, 25k build minutes, 2M function invocations.
  • Anthropic Tier 1 — upgrade before launch (memory note: 10K out tokens/min cap will bite).
  • Plaid production, pay-per-active-item. No tier.

v2 1k–5k users

  • Supabase — watch egress (250GB included). Add connection pooler config.
  • Netlify Pro ($20) already in place — watch build minutes if CI gets heavier.
  • PostHog — 1M events/mo free; add event sampling for heavy users.
  • Resend Pro ($20) if email volume > 3k/mo.
  • Anthropic Tier 2 auto-unlocks at $400 spent + 7d age.

v3 5k–25k users

  • Supabase Team ($599) for SOC2 + log retention + point-in-time recovery.
  • Plaid negotiate volume pricing — talk to AE at 5k linked items.
  • Cloudflare in front of Netlify for cache + WAF (free tier covers it).
  • Anthropic prompt caching already in place per memory — verify cache hit rate stays > 70%.

v4 25k+ users

  • Supabase read replicas + dedicated compute.
  • Stripe negotiate interchange-plus.
  • Re-evaluate self-hosting AI inference if Anthropic spend > $20k/mo.
  • Postgres connection pool via PgBouncer/Supavisor mandatory.
  • Hire SRE or pull in DevOps contractor.

Bottlenecks to watch these bite before you hit "scale"

AnthropicURGENT
Tier 1 caps at 10K output tokens/min. Coach + carousel + blog generation share this budget. One viral signup spike = throttling. Action: upgrade tier before launch (memory note 2026-05-12).
SupabaseSOON
Default connection pool = 60. At ~5k DAU with idle-keep-alive, you'll exhaust it. Action: enable Supavisor transaction-mode pooler before 1k DAU.
PlaidSOON
Per-item monthly fee + per-API call. Cost scales linearly with user count, not request volume — so the cap is "how many users", not "how active are they." Action: track $/linked-item monthly; renegotiate at 5k items.
StripeFIXED
2.9% + 30¢ on every charge. On $14.99/mo: ~$0.74 per user per month (~5% of revenue). On $129/yr: ~$4.04 once (~3.1%). Tiny but bakes into margin — already in vendor table.
PostHogWATCH
Free tier = 1M events/mo. A noisy event ("scroll", "input") on 5k DAU burns it fast. Action: audit event volume monthly; drop any event you'd never query.
NetlifyLATER
Bandwidth = the limiter, not function invocations. Static HTML + WebP images keep this low. Won't bite until 10k+ DAU. Cloudflare in front later if needed.

Update protocol

Weekly (Sundays, 15 min):
  • Pull user count from Supabase: select count(*) from auth.users where deleted_at is null;
  • Pull paying users from Stripe → Customers (filter active subscriptions).
  • Update Inputs at top — everything below recomputes.
Monthly (when invoices land):
  • Update each vendor's Monthly cost column with the actual invoice.
  • If a vendor's $/user is rising faster than revenue/user, flag it — that's a margin leak.
  • Export this page to PDF (Cmd-P → Save as PDF) for monthly investor update.
Don't automate yet. Manual updates force you to look at the numbers. Build API pulls only when this takes >30 min/week.
