## Frontmatter schema

Every file opens with YAML frontmatter containing:

```yaml
title: Page title
slug: url-slug
url: /cluster/slug/
meta_title: SEO title tag
meta_description: SEO meta description
target_keyword: Primary target keyword
secondary_keywords:
  - list of secondary keywords
cluster: budgeting | taxes | business
page_type: pillar | supporting
parent: /cluster/  (for supporting pages only)
internal_links:
  - list of URLs this page links to
cta: CTA button text
```

## Voice rules enforced on every page

- No em dashes in customer-facing content
- Short sentences
- No jargon
- Customer is hero, Able is guide
- Identity reinforcement via "I am able to..." language
- Transformation section near end of every piece
- CTA block with "Start Free Trial" button

## Internal linking strategy

Every page links back to its cluster pillar. Every pillar links to all its supporting pages. Cross-cluster links happen when topics genuinely intersect (e.g., budgeting pillar links to taxes pillar, business pillar links to budgeting pillar).
